# schattdecor Sense 小程序演示版

这是可直接用于微信开发者工具验收的前后端发布包，包含：

- 微信小程序用户端 `miniprogram/`
- 可本地运行的轻量后端 `server.py`
- 独立后台查看页面 `admin_web/`
- HarmonyOS Sans SC Regular / Medium / Bold 字体 `assets/fonts/`
- 页面展示使用的扫描花色素材 `demo_data/decor_info/`
- 项目说明文档 `docs/`

## 发布范围

本发布包用于界面与业务流程测试，不包含 DINOv3 模型、特征库、实拍参考数据、训练代码或评测代码。图片识别接口由轻量后端返回可测试结果，搜索、花色详情、收藏、浏览记录、PDF、客服数字对话及后台页面均可在本地体验。

当前随包提供本地已存在的 `49` 类扫描花色素材。目标集合中的 `14-18101-004` 在现有工作目录中只有实拍参考图，没有扫描版素材，因此未放入本发布包；获得其扫描版后应补充到 `demo_data/decor_info/14-18101-004/`。

## 快速运行

1. 在仓库根目录启动后端：

```powershell
python server.py
```

若命令立即退出且没有任何服务输出，请确认 `python --version` 指向真实 Python 3 环境，而非 Windows Store 占位程序；也可使用本机实际 Python 可执行文件路径运行 `server.py`。

2. 可选：使用 Node.js 检查小程序文件和扫描素材是否完整：

```powershell
npm run check:miniapp
```

3. 确认后端可访问：

```text
http://127.0.0.1:8000/health
```

4. 在微信开发者工具中导入：

```text
<本仓库目录>\miniprogram
```

5. 在开发者工具的“本地设置”中启用“不校验合法域名、web-view 域名、TLS 版本以及 HTTPS 证书”，点击“编译”。

如果微信开发者工具提示当前 AppID 无权限，可在导入项目时选择自己的测试 AppID 或小程序 AppID。

## 重要提醒

- `demo_data/decor_info/` 中是界面展示素材，不是模型训练数据。
- 发布到公开 GitHub 仓库前，请确认扫描花色素材与 HarmonyOS Sans SC 字体许可适用于公开分发。
- 正式上线时应将 `server.py` 替换为具备持久化、鉴权与正式识别能力的生产后端，并使用 HTTPS 合法域名。
