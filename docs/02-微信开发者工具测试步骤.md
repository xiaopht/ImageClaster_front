# 微信开发者工具测试步骤

## 测试前准备

本发布包不需要模型或特征库。页面数据由根目录的 `server.py` 读取 `demo_data/decor_info/` 中的扫描花色素材后提供。

准备环境：

- Python 3.10 或更高版本；
- 已登录的微信开发者工具；
- 下载后的完整仓库目录。

## 启动本地后端

在仓库根目录执行：

```powershell
python server.py
```

如果 `python --version` 指向 Windows Store 占位程序，命令可能会无输出直接结束。此时请安装 Python 3，或用本机实际 Python 可执行文件的完整路径执行 `server.py`。

运行成功时终端显示：

```text
Xiaote mock backend running on http://127.0.0.1:8000
```

如 `8000` 端口正在被其他本地服务使用，可在单独检查发布版后端时执行 `$env:XIAOTE_PORT = "8011"` 后再运行 `python server.py`。正式使用小程序测试时，应保证发布版服务运行在默认 `8000` 端口，或同步修改小程序 API 地址。

浏览器打开以下地址应返回健康状态，其中 `patterns` 应为 `49`：

```text
http://127.0.0.1:8000/health
```

后台测试页面地址为：

```text
http://127.0.0.1:8000/admin-web/
```

## 导入小程序

1. 打开微信开发者工具。
2. 选择“导入项目”。
3. 项目目录选择：

```text
<仓库下载目录>\miniprogram
```

4. 如果当前账号不能使用项目配置中的 AppID，请改选自己的小程序 AppID 或测试号。
5. 打开“详情 / 本地设置”，勾选“不校验合法域名、web-view 域名、TLS 版本以及 HTTPS 证书”。
6. 点击“编译”。

小程序默认后端地址已经是：

```text
http://127.0.0.1:8000
```

## 页面验收流程

1. 首页搜索 `57204` 或 `Amalfi`，确认能显示扫描花色卡片。
2. 点击花色卡片打开大图，确认能看到图片、名称、编号、收藏按钮和左上角 `PDF` 按钮。
3. 点击收藏星标，进入“我的 / 我的收藏”，确认同一花色显示为已收藏状态；再次点击应取消收藏。
4. 从首页或收藏打开花色详情，检查“浏览记录”中新浏览花色位于首位；从浏览记录中再次打开同一详情时不应改变记录顺序。
5. 在“我的”页切换 `EN / 中文`，确认花色分类和客服页文字随语言变化。
6. 在首页点击客服浮钮，确认能进入客服页；关闭圆点应能隐藏首页浮钮。
7. 在客服输入栏输入职业编号 `1` 并发送，再输入区域编号 `4` 并发送，确认出现联系方式卡片。
8. 点击客服卡片中的“仅复制手机号”和“复制完整信息”，确认开发者工具提示复制成功。
9. 客服对话完成后输入 `0`，确认流程重新回到职业选择。
10. 打开后台页面，确认测试操作产生的事件或反馈不会直接展示在小程序用户页面中。

## 本发布包的限制

- 拍摄或上传图片后的识别结果是界面联调用模拟结果，不代表真实模型输出。
- 当前提供 `49` 类正式扫描素材；`14-18101-004` 缺少扫描版，因此不会显示在页面候选中。
- 正式上线前需要替换为 HTTPS 后端、生产鉴权与真实识别服务。
