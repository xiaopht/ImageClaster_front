# 小程序 UI 样式调整入口

本说明只覆盖最终呈现相关代码。修改前建议在微信开发者工具中保留一个可回退版本，修改后点击“编译”立即查看效果。

## 最重要的入口

| 文件 | 控制内容 | 主要可调位置 |
| --- | --- | --- |
| `app.wxss` | 所有页面共用的颜色、字体、按钮、花色卡片、收藏星标、大图弹层、反馈/联系弹窗、首页客服浮钮外观 | 顶部 `page` 变量；`.pattern-*`；`.detail-*`；`.service`；`.feedback-*`；`.lead-*` |
| `pages/index/index.wxml` | 首页元素的出现顺序与绑定内容 | 搜索框、上传区、分类、结果卡片、客服浮钮、大图及弹窗结构 |
| `pages/index/index.wxss` | 首页最终布局 | `.home-content`、`.upload-box`、`.segment*`、`.mode-*`、`.start-button`、首页 `.pattern-*` 覆盖 |
| `pages/index/index.js` | 客服浮钮位置行为与首页交互 | `SERVICE_FLOAT_STYLE` 调整客服浮钮尺寸、初始位置和可拖边界 |
| `pages/camera/camera.js` | 底部“拍照”快捷入口桥接页 | `openHomeCaptureShortcut()` 将用户送回首页并触发首页上传区域；该页不再作为独立呈现页面调整 |
| `pages/mine/mine.wxml` / `.wxss` | 我的页、收藏/浏览记录、登录注册弹窗 | `.mine-hero`、`.profile-*`、`.mine-panel .pattern-*`、`.auth-*` |
| `pages/service/service.wxml` / `.wxss` | 客服会话页 | `.service-header`、`.message-*`、`.choice-*`、`.chat-input`、`.service-tabbar` |

## 共用样式的修改原则

- 页面底色、品牌绿、正文色、收藏黄色和字体集中在 `app.wxss` 的 `page` 变量中；在这里改一次会影响多个页面。
- 卡片小图、分类标签、收藏按钮、名称编号和“更多颜色”位于 `app.wxss` 的 `.pattern-*`、`.star` 和 `.more-button`。首页和我的页会在自己的 `.wxss` 中进一步覆盖尺寸。
- 打开花色后的大图展示严格由 `app.wxss` 的 `.detail-*` 控制，包括左上角 PDF、右上角关闭、主图舞台、左右翻页、收藏按钮及底部名称编号。
- 首页可拖动客服按钮的颜色、圆角、宽高在 `app.wxss` 的 `.service`；初始位置和拖动边界在 `pages/index/index.js` 的 `SERVICE_FLOAT_STYLE`。调整宽高时应同步修改两处。

## 不能直接写注释的呈现配置

`app.json`、各页 `.json` 是微信要求的严格 JSON 文件，直接加入注释会导致项目无法编译。因此不要在这些 JSON 中插入说明文字。

| `app.json` 字段 | 显示效果 |
| --- | --- |
| `window.navigationBarTitleText` | 原生导航栏标题 |
| `window.navigationBarBackgroundColor` / `navigationBarTextStyle` | 原生导航栏底色与文字明暗 |
| `window.backgroundColor` | 页面加载或下拉时的原生背景 |
| `tabBar.color` / `selectedColor` / `backgroundColor` | 微信原生底栏文字色、选中色和底色 |
| `tabBar.list[*].text/iconPath/selectedIconPath` | 底栏三个入口的文字及图标 |

## 文案、字体和图片资源

| 文件/目录 | 用途 |
| --- | --- |
| `config.js` | 品牌名、接口地址、部分全局显示配置，以及客服浮钮以外的业务常量 |
| `utils/i18n.js` | 中文和英文页面文案、花色分类显示文本；未来新增德语应在此扩展语言表 |
| `fonts/` | `HarmonyOS Sans SC` 字体文件，由 `app.js` / `config.js` 加载 |
| `assets/logo/`、`assets/tabbar/` | Logo 和微信底栏图标 |

## 在微信开发者工具中调整和验证

1. 导入项目目录 `D:\夏特项目\ImageClaster\部署 4.0\miniprogram`。
2. 本地后端运行在 `http://127.0.0.1:8000` 时，开发工具详情设置中勾选“不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书”。
3. 修改 `.wxss` 或 `.wxml` 后点击“编译”；打开首页、拍摄、我的、客服页逐页查看。
4. 首页拖动绿色客服小按钮，确认能移动且轻点仍能打开客服页。
5. 搜索或识别得到花色后，检查小图、收藏黄色态及打开大图后的 PDF、收藏、翻页布局。

## 本地花色如何显示

小程序页面不会凭空生成花色。首页发起搜索或图片识别后，`utils/api.js` 通过 `config.js` 中的 `apiBase` 请求本机后端；后端只返回本地花色目录或本地数据库能找到的数据。返回的名称、编号、分类及 `image_url` 经 `formatPatternItems()` 转为页面字段，再由花色卡片的 `<image src="{{item.imageSrc}}">` 和文本节点显示。

因此，在微信开发者工具中看到的是本机后端当前提供的数据；如果某张花色仅存在于本地且后端已将它纳入花色目录或检索结果，它就可以在本地预览中显示，但不会自动出现在线上小程序或其他人的设备上。
