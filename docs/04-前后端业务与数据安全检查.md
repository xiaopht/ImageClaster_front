# 前后端操作逻辑风险检查

> 说明：本文记录正式业务后端的设计与风险检查结论。本 GitHub 无模型发布版使用 `server.py` 提供本地交互演示，测试数据仅驻留在进程内存中；正式持久化与角色权限逻辑不作为该演示服务的上线实现。

检查范围：用户注册、反馈存储、前后端连接、中英文切换、辅助分类匹配、浏览记录、PDF 导出入口。不包含模型算法准确率评估。

## 已检查并已处理

1. 用户注册与登录
   - 注册信息写入 `data/platform/xiaote_platform.sqlite3` 的 `users` 表。
   - 密码使用 `PBKDF2-SHA256` 加盐哈希，不明文保存。
   - 登录后写入 `sessions` 表，小程序请求通过 `Authorization: Bearer <token>` 连接后端。
   - `sales/admin` 角色注册需要内部访问码，后台接口要求 `sales/admin` 权限。
   - 已把语言字段限制为 `zh-CN/en-US/de-DE`，保留未来德语扩展入口。

2. 用户反馈与行为数据
   - `/api/feedback` 写入 `feedback` 表，反馈图片另存到 `data/platform/feedback_images`。
   - `/api/events` 写入 `events` 表，后台网站通过 `/api/admin/*` 独立查看，数据不放在小程序页面中。
   - 后台接口已有角色保护。

3. 浏览记录
   - 打开花色大图时调用 `/api/patterns/{pattern_id}` 记录浏览。
   - 浏览记录接口现在只返回花色卡片，不返回搜索、收藏、识别等操作记录。
   - 重复浏览同一花色会删除旧浏览记录，再把新记录放到首位。

4. 前后端连接
   - `miniprogram/utils/api.js` 统一管理 `apiBase`、token、上传识别、PDF 下载和浏览记录写入。
   - mock 后端与正式 FastAPI 后端都补齐了浏览记录、语言偏好和辅助分类测试逻辑。

5. 中英文切换
   - 新增 `miniprogram/utils/i18n.js`，中文/英文页面文案集中维护。
   - “我的”页语言按钮会切换 `zh-CN/en-US`，并同步到本地存储和 `/api/user/preferences`。
   - TabBar 文案会随语言更新。
   - 后续添加德语时，可在 `i18n.js` 增加 `de-DE` 字典并加入切换顺序。

6. 辅助分类匹配
   - 小程序增加“不使用辅助”选项，默认不限制分类。
   - 文字搜索会把已选分类传给 `/api/patterns`。
   - 图像识别后端会先在选中分类内过滤候选，再返回 Top 结果；开启辅助分类后，不会返回分类外花色。

7. 大图与 PDF 入口
   - 首页、拍摄页、收藏、浏览记录打开大图时统一使用 `AI识图_01-5.jpg` 风格：深色遮罩、超大圆角图、右上关闭、左右切换、底部收藏和“更多颜色”、图下名称/编号/序号。
   - PDF 按钮已放在大图弹层左上角，当前复用收藏夹 PDF 导出接口 `/api/favorites/export.pdf`。

## 仍建议上线前加固

1. 注册字段校验：当前用户名和密码有长度限制，但手机号、邮箱格式还可以进一步校验。
2. 反馈图片大小：当前可保存 base64 图片，生产环境建议限制图片大小、格式和单用户提交频率。
3. HTTPS 与合法域名：小程序上线必须把 `apiBase` 改为 HTTPS 域名，并在微信公众平台配置 request/upload/download 合法域名。
4. 访问频率：注册、登录、反馈、识别上传建议增加限流，防止接口被刷。
5. PDF 语义：当前导出的是收藏夹 PDF；如果需要“当前花色单页 PDF”，应新增单花色 PDF 接口。
6. 分类数据质量：辅助分类硬过滤依赖 `info.json` 中的 `textureName/tags`，上线前应抽查花色目录的分类标注是否完整。
