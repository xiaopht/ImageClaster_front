# 发布版结构与运行说明

## 目标

本目录是前端和后端业务演示发布版。下载后无需配置模型与特征库，启动轻量后端即可在微信开发者工具中查看页面、搜索扫描花色并操作用户流程。

## 目录结构

```text
schatt_ai_wechat_app/
├─ miniprogram/              微信小程序页面、交互和配置
├─ admin_web/                与小程序分离的后台查看页面
├─ assets/fonts/             页面使用的 HarmonyOS Sans SC Regular / Medium / Bold 字体
├─ demo_data/decor_info/     扫描版花色展示素材
├─ docs/                     统一命名的说明文档
├─ server.py                 无模型本地测试后端
├─ tools/check-miniprogram.js 小程序与素材完整性检查
├─ package.json              检查命令入口
├─ .gitignore                发布排除规则
└─ README.md                 入口说明
```

## 已包含的功能

- 首页文字搜索、花色列表和大图详情。
- 拍摄/上传后的演示识别结果流程。
- 收藏、取消收藏与浏览记录。
- 单花色 PDF 展示流程。
- 中英文界面切换。
- 客服页数字对话：输入职业编号、地区编号后展示联系方式。
- 后台页面对测试过程中产生的事件、反馈与线索的查看入口。

## 未包含的模型能力

此发布包不包含：

- DINOv3 模型权重；
- 预计算特征库；
- 实拍参考数据；
- 模型训练、特征生成与评测代码。

因此，`server.py` 的识别结果用于界面和前后端联调，不代表真实模型准确率。

## 启动方式

后端仅依赖 Python 标准库：

```powershell
cd "<仓库下载目录>"
python server.py
```

运行成功后服务地址为：

```text
http://127.0.0.1:8000
```

随后在微信开发者工具中导入 `<仓库下载目录>\miniprogram`。小程序默认请求 `http://127.0.0.1:8000`，不需再改接口地址。

如需在不占用 `8000` 端口的情况下单独检查发布后端，可临时启动：

```powershell
$env:XIAOTE_PORT = "8011"
python server.py
```

导入微信开发者工具前可运行完整性检查：

```powershell
npm run check:miniapp
```

## 正式上线替换点

- 将本地 HTTP 地址替换为已备案且加入微信合法域名的 HTTPS API 域名。
- 将内存测试数据改为生产数据库及合规登录鉴权。
- 将 mock 识别接口替换为模型服务或独立识别服务。
- 在获得完整授权后补充/分发扫描素材和字体资源。
- 补齐缺少的 `14-18101-004` 扫描花色素材。
