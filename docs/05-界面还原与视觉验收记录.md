## 4. 字体方案

`.ai` 设计源文件记录的字体为 HarmonyOS Sans SC，实际使用 Regular / Medium / Bold 三个字重：

- `HarmonyOS_Sans_SC_Regular.ttf`
- `HarmonyOS_Sans_SC_Medium.ttf`
- `HarmonyOS_Sans_SC_Bold.ttf`

加载方式：`wx.loadFontFace`，配置集中在 `miniprogram/config.js` 的 `FONT_CONFIG`。字体由本地后端 `/fonts/` 静态托管，源文件位于 `assets/fonts/`。全局 WXSS 通过字体变量引用。字体文件较大，正式发布前建议子集化或 CDN 加载。

## 5. 视觉自动化方案

命令：

```powershell
npm run assets:ui
npm run visual:prepare
npm run visual:capture
npm run visual:compare
```

截图策略：

- tabBar 页面通过 `switchTab` 打开，避免 `reLaunch` 导致真实 tab 状态丢失。
- 视觉状态通过页面方法 `applyVisualState(screen.id)` 注入，不依赖真实登录、真实 AI 接口或随机数据。
- 微信开发者工具原生截图缺少系统 tabBar，因此截图脚本临时打开 `visualSyntheticTabbar`；截图结束后自动关闭，避免人工预览出现两行底栏。
- 如果原生截图超时，脚本保留桌面截图兜底。
- 截图时同步输出关键元素尺寸/位置到 `runtime-layout-report.json`，用于判断差异来自布局、裁剪还是截图环境。
